# L7 - CSS Flexbox 

with float and clear , aligning content was hectic and often buggy, CSS Flexbox makes it easier to align the content 
display:flex lets us utilize the properties of flexbox and lets us manipulate the posetioning of the elements inside a container

Display flex aligns the contents horizontally irrespective of them being block or inline elements.(by default)
justify-content: lets us change the orientation of the content in direction of main axis (works on main axis)
align-content: lets us change the orientation of the content in direction of cross axis (works on cross axis)

(NOTE - THE ORIENTATIONS ARE ONLY IN EFFECT INSIDE THE CONTAINER , NOT ON THE CONTAINER ITSELF)

The Items inside flex box are layed in the "flex-dircrion"  , by default left -> right (row) , but can be changed to top to bottom (column)
the flex-direction:row; aligns items from left to right by default

IMPORTANT - 
main axis , is the direction in which the elements are alligned , if row then horizontal , if column then vertical
cross axis is perpendicular to main axis , when we change the flex direction , theses axis also interchange
main start , main end , cross start , cross end - > starting and ending points of main and cross axis .
main size , cross size -> distance from main start to main end and cross start to cross end

flex-wrap:wrap/wrap-reverse; -> if the content of the flex overflows , it will start a new row/column inside the flexbox that contains thee overflowing contents (wrap reverse starts from bottom instead of top)

align-content: property used when we use wrap instead of align-items as it does not work with wraps as intended

flex-flow: <flex direction> <flex wrap> ; - lets us define 2 properties at once , flex direction and flex wrap

gap:30px; will add gap between the elements even without margin
gap: 30px 40px; will change row-gap to 30px and column-gap to 40px

item property - 
order: 2; by default order value of each flexbox element is 0 , the greater the value , the more end the element will be placed in wrt other elements
          order lets us place elements in specific order even if defined defferently in html 

flex-grow : 1; if difined in a single class of multiple elements , all of them will together grow to cover the width of the container
               if defined in one element inside container , flex-grow:2; means the element will have twice the size of other elements inside the containers
               default value 0 , ie - will not fill container

flex-shrink :2; is item property , will shrink the element twice the speed of normal element when needed (window size change nd all) 

align-self: used to move a single element while keeping other elements as is .

flex-basis: sets the initial, default size of a flex item before the remaining space in the container is distributed

# L8 - CSS Grid

display:grid; - keeps the container as is but we can explicitly define the number of rows and column we want

grid-template-columns: [name1]100px [name2]300px; -> defines that the grid has 2 columns first 100px , second 300px, and the content aligns in direction of rows AND number of rows can increse as elements increase but columns remain same 

([name1] and [name2] are optional , they are used to give names to the starting grid line of the row or column)

eg - 
1   2
3   4
5


grid lines are seperations between the indevidual cells of the grid
grid calls are individual sells inside a grid , a cell is like a box inside the grid
grid track is the area between any 2 directly next to each other parellel grid lines
grid area  is the area between any 4 grid lines

we can give names to grid lines , and they can also be accessed via indexing 

eg - box  box  box |  1 or -3
     box  box  box |  2 or -2
     box  box  box |  3 or -1
     --------------
      1    2    3
      or   or   or
     -3   -2    -1

grid-row:start/end; ,grid-column:start/end; (where start and end are index values or name of the grid line we want to cover by that grid element's cell) - when we want to place an element at a certain place in the grid . (will push the elements to 
place the element in their place and the whole orientation shifts accordingly) , to overlap , give the same grid-row/column to the element we want to overlap.

rest of the cells shift logically.

CSS Grid - 2 
Instead of naming grid lines , we can name grid AREAS
(in parent)

grid-template-areas : "nav nav nav"  
                      "side article article"
                      "footer footer";  (assigns based on cells)

(in the element)
grid-area:nav; (will give all the nav area to the given element)

in this we can overlap as well by giving the element the same grid-area


-- fr Unit -- means Fraction
used to depict the fraction of area we wish to dedecate to elements wrt the available width (in parent using grid-template-row/column)
look at the code and contemplate the relation between areas and template comlumns in fr

(the propertes below are applied inside the indevidual cells of the elements only)
justify-items:center;  will set the elements to be at center of their respective cells (horizontally) - defined in container
align-items:center; will set the elements to be at center of their respective cells (vertically) - defined in container
item property - align-self:start/end/center; - will allign the desired element of the grid only.  - defined in element

CSS repeat(num_repetitions , value to repeat) used when we have to repeat the values
minmax(min_possible_value , max_possible_value) is defined such that one of the values is variable and other one acts as a bound


WHOLE CONTAINER SPECIFIC PROPERTIES 
in cases where the grid is smaller than the container itself , we  can align the whole grid with respect to the container with the following properties
justify-content:center/end/start;  aligns the grid to the center horizontally
align-content:center/end/start;  aligns the grid to the center vertically
place-content:center; aligns the grid both horizontally and vertically.
